# Unity - From Zero

- Ни одна сцена в билде не должна содержать консольных ошибок. Ограничивать платформенно-специфичные вызовы через препроцесс-директивы (#if UNITY_PS4, #if UNITY_PS5), чтобы исключить ошибки компиляции при смене платформы. Удалять лишние логи. Оставлять только минимальный Debug.Log, отражающий последовательность ключевых вызовов
- Настроить сериализацию Unity, .gitignore и при необходимости .gitattributes. Ни один коммит не должен содержать артефакты или файлы из несвязанной ветки. Коммиты должны быть разделены по контексту
- Использовать только ассеты с поддержкой HDRP и URP. Сохранять исходные материалы, декали и шейдеры для быстрого переключения рендер-пайплайна
- Разделять коллизию и LOD в окружении. Использовать LOD только при необходимости. Коллизии по возможности строить из примитивов и разделять физику по слоям
- Использовать композитные префабы везде, где это возможно (включая UI). Не изменять параметры префабов на сцене. Создавать модульные префабы с связями через ScriptableObjects или абстракции. Разбивать префабы на части, если в них более 8 дочерних объектов. По возможности использовать стандартные компоненты Unity вместо кастомных решений
- В папке готового проекта не должно быть лишних файлов. В коде не должно быть лишних комментариев или неиспользуемых методов
- По возможности не использовать Odin Inspector и DI через Zenject (из-за замедления загрузки в Nintendo slow-mode)
- UI должен быть полностью динамическим для кроссплатформенной замены.Настроить TMP-ассеты под каждый шрифт и вес. Сохранить ссылку на Figma-дизайн
- Использовать Input System. При необходимости создавать несколько action maps (Player/UI) и переключать их в зависимости от состояния игры. Использовать одну EventSystem на сцену
- Вся релевантная информация по проекту (дизайн-идея, известные и нерешённые баги) должна передаваться продакт-менеджеру вместе со ссылкой на GitHub/ветку
